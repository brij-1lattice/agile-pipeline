# Reference — INDEX.md generation

> Single source of truth for `{index_path}`. **`manage-stories index` is the sole writer** — it
> regenerates the whole file from the story frontmatter. `analyze-sprint` and `execute-sprint`
> regenerate via this same flow after a run; the four QA-chain skills (`code-review-sprint`,
> `generate-test-sprint`, `qa-sprint`, `verify-sprint`) **do not** touch `{index_path}` (they run
> in parallel sessions and would race on it) — they stamp only their own stories' frontmatter and
> ask the user to run `/manage-stories index` when convenient. Output must be **idempotent** —
> running it twice produces zero diff.

Read every `{stories_dir}**/story-*.md` (exclude `{archive_dir}`; backlog stories go in their
own section, not in sprint sections). Parse YAML frontmatter and count tasks. Write
`{index_path}` using this template. Sprint labels come from the config `sprint_labels` map.

**Multi-operator mode** (any `.claude/operators/*.md` exists, per `operator-profile.md`): group
**operator → sprint → topic** instead of sprint → topic. Each operator gets a `## Operator <name>`
section; within it a `### <name>/N — <goal>` heading per sprint (goal from the operator's profile
`Sprints` table), then the same `#### <topic>` tables and columns below. The Summary's **Owners**
line shows each operator's active sprint + branch. Everything else (Backlog section, column
meanings, idempotence) is unchanged. With **no** `operators/` dir, use the sprint → topic layout
below verbatim.

```markdown
# Stories Index

> Generated by `/manage-stories index`. Do not hand-edit — changes overwrite.

**Summary:** <N> stories · <SP> SP · <verified> verified · <tested> tested · <tests-generated> tests-generated · <code-reviewed> code-reviewed · <done> done · <in-progress> in-progress · <ready> ready · <draft> draft · <blocked> blocked · <deferred> deferred (backlog)
**QA failed:** <code-review-failed> code-review-failed · <testing-failed> testing-failed · <verification-failed> verification-failed (each carrying open feedback items)
**Owners:** <owner> (<count>) · <owner> (<count>) · …

…

## Column meanings

- **Tasks:** `<completed + cancelled> / <total>`. A story is eligible for `done` only at `N/N`, N > 0.
- **Owner:** person responsible for execution + testing. Empty → lint `unassigned`.
- **Design:** linked filename basename, `none-needed`, or `→ story-<id>` for follows-template.
- **Depends on:** comma-separated story IDs. Empty = no dependencies.
- **CR / Gen / Test / Verify:** the four QA-gauntlet stages. Each column is a pure function of the
  single linear `status` (which the builder resets to `done` — clearing the stage-date stamps — on
  every re-build, so the board never shows a stale ✓ for code that changed since). `✗N` counts the
  **open blocking** `- [ ]` items in that stage's section (`[true]` A/B/C; `> nit`/`> justified`/`>
  choice` notes are non-blocking and not counted). The full mapping — every status, every column:

  | `status` | CR | Gen | Test | Verify |
  |---|---|---|---|---|
  | `draft` / `ready` / `in-progress` / `blocked` / `deferred` | — | — | — | — |
  | `done` | — | — | — | — |
  | `code-reviewed` | ✓ | — | — | — |
  | `code-review-failed` | ✗N | — | — | — |
  | `tests-generated` | ✓ | ✓ | — | — |
  | `tested` | ✓ | ✓ | ✓ | — |
  | `testing-failed` | ✓ | ✓ | ✗N | — |
  | `verified` | ✓ | ✓ | ✓ | ✓ |
  | `verification-failed` | ✓ | ✓ | ✓ | ✗N |

  (A `*-failed` story carries ✓ for the gates it already cleared this cycle — `verification-failed`
  shows ✓ ✓ ✓ ✗N — because it genuinely passed them before failing the last stage. Gen has no failed
  state of its own; a generation blocker shows as `status: blocked`, not a Gen ✗.)
```

Rules:
- Sort within each topic by sequence number ascending.
- Backlog stories (status `deferred`, living in `{backlog_dir}`) appear **only** in the Backlog section — excluded from every sprint's section and from sprint story/SP totals. Add the `deferred` count to the Summary line.
- Always write the "do not hand-edit" warning comment.
- Idempotent: a second `index` run yields zero diff.
